## 2026-07-20 — Fix: Google Sheet export "The caller does not have permission"

Root cause: export-sheets uses the DB-stored google_refresh_token (getGoogleAccessToken),
never the browser token. That token was minted with the old `drive.readonly` scope (before
commit a658808 upgraded the scope to full `drive`). Refreshing never upgrades scopes, so the
Sheet is created OK (spreadsheets scope) but the Drive move-into-folder is denied. Re-linking
didn't help: DriveAuthButton "disconnect" only cleared localStorage — never the DB token — and
reconnect didn't force a fresh grant, so Google returned no new refresh token.

Changes:
- NEW app/api/auth/google-drive/disconnect/route.ts — revokes the grant at Google + nulls
  app_users.google_refresh_token, so the next connect is a fresh grant with full `drive` scope.
- DriveAuthButton.tsx — disconnect now POSTs to that endpoint before clearToken().
- driveToken.ts — added `select_account: true` so the user can pick the folder-owning account.
- export-sheets/route.ts — added tokenScopes() helper; the folder-move failure message now
  distinguishes a missing drive-write scope (→ reconnect) from a wrong-account/Shared-Drive case.
Immediate unblock for existing users: revoke app at myaccount.google.com/permissions, then
disconnect + reconnect in the app.

## 2026-06-10 — Fix v2: Logic Rewrite (JM-FORM-SUMMARY-logic-flow.md)

### Critical bugs fixed:

1. **Gold price formula** (pricing.ts) — was using pre-derived $/gram rates from `daily_metal_rates`.
   Now: raw 24K spot ($/oz) + loss% per karat via `goldPricePerGram(loai_vang, nvl)`.
   Formula: 18K = `gold_24k × 1.06 × (18/24) / 31.103`, PT = `pt_price × 1.17 / 31.103`, etc.

2. **Diamond weight formula** (fix_v2_logic.sql) — GENERATED column `weight_gr` was `weight_ct_after × 0.2`.
   Now: `weight_ct_before × 0.2` (= tl_truoc_xu_ly_ct / 5 per JM-FORM).
   Same fix applied to `total_price` GENERATED: `weight_ct_before × unit_price_per_ct` (= t_gia_xoan).

3. **Von SX formula** (pricing.ts `calcVonSanXuat`) — template-aware:
   - CH1/CH2: Σt_gia_xoan + Σt_phi + tien_vang + gia_cong + duc + thiet_ke + resin + phi_phu_kien
   - ADM: Σt_gia_xoan + Σt_phi + tien_vang (no individual fees)
   - CH1_AG3/VNSI_AG3: tien_vang only

4. **CIF formula** (pricing.ts `calcCIFPrice`) — template-aware:
   - CH1/ADM/CH1_AG3: purchase × 1.05
   - VNSI_AG3: purchase × 1.10
   - CH2: null (no CIF column)

5. **Template support** — added `template_type` + NVL snapshot columns to `invoice_headers` (migration).
   All 6 API routes updated to read `template_type, nvl_*` from invoice instead of fetching `daily_metal_rates(*), pricing_rules(*)`.

6. **UI labels** — fixed to match SUMMARY sheet:
   - "HPUSA" → "Vốn SX"
   - "HP CIF" → "CIF/SP"
   - "No-Gem Wt" → "T.Phẩm vàng TT"
   - "HP Tag/FB" → removed
   - Gem table: added Mã Xoàn, Size Range, TL Trước/Sau, and "auto" annotations
   - CH2 template hides CIF column in JMFormView
   - ADM template hides individual gia công fields in ItemCard
